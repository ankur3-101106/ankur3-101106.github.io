Lecture 51:
Cryptoanalysis and Stream Cipher​

Stream Cipher (LFSR Model)​

  • Keystream:

    zt=f(st)z_t = f(s_t)zt​=f(st​)

  • Encryption:

    ct=pt⊕ztc_t = p_t \oplus z_tct​=pt​⊕zt​

  • Decryption:

    pt=ct⊕ztp_t = c_t \oplus z_tpt​=ct​⊕zt​

LFSR Structure​

  • State:

    sts_tst​

  • Linear recurrence:

    st+1=∑aist−is_{t+1} = \sum a_i s_{t-i}st+1​=∑ai​st−i​

✔ Defined by feedback polynomial

Problem​

  • LFSR is linear ❌ Easily breakable

Solution​

  • Add non-linear function: zt=f(st)z_t = f(s_t)zt​=f(st​)

✔ Improves security

Algebraic Attack​

Idea​

  • From:

    zt=ct⊕ptz_t = c_t \oplus p_tzt​=ct​⊕pt​

  • If plaintext known:

    zt knownz_t \text{ known}zt​ known
  • Also: zt=f(st)z_t = f(s_t)zt​=f(st​)

Build Equations​

  • For multiple time steps: f(st)=ztf(s_t) = z_tf(st​)=zt​

✔ System of equations

Goal​

  • Solve for: st⇒recover keys_t \Rightarrow \text{recover key}st​⇒recover key

Problem​

  • Equations are non-linear

Linearization​

  • Replace products:
xixj=yijx_i x_j = y_{ij}xi​xj​=yij​

Example​

x1x2→y12x_1 x_2 \rightarrow y_{12}x1​x2​→y12​ x1x3→y13x_1 x_3 \rightarrow y_{13}x1​x3​→y13​

Result​

  • Convert system into: linear equations in yij\text{linear equations in } y_{ij}linear equations in yij​

✔ Easier to solve

Re-linearization​

  • Use identities like:
x1x2x3x4x_1 x_2 x_3 x_4x1​x2​x3​x4​
  • Rewrite as combinations:
(x1x2)(x3x4)(x_1 x_2)(x_3 x_4)(x1​x2​)(x3​x4​)
  • Substitute: y12,;y34y_{12},; y_{34}y12​,;y34​

Key Insight​

  • Non-linear → transformed into linear system
  • Solve using linear algebra

Limitation​

  • Number of variables increases rapidly
  • Needs enough equations

Final Summary​

  • LFSR → linear → weak

  • Add non-linearity → stronger

  • Algebraic attack:

    • Build equations
    • Linearize
    • Solve system

✔ Recovers internal state / key

Lecture 52:
Modern Stream Ciphers​

Modern Stream Cipher Structure​

Basic Working​

  • Keystream:

    ztz_tzt​

  • Encryption:

    ct=pt⊕ztc_t = p_t \oplus z_tct​=pt​⊕zt​

Initialization​

  • Key + IV → initial state:

    s0s_0s0​

  • Run cipher for some rounds (warm-up)

✔ Removes initial bias

LFSR + Non-linearity​

  • Linear part:

    LFSR\text{LFSR}LFSR

  • Non-linear part:

    NLFSR / Boolean functions\text{NLFSR / Boolean functions}NLFSR / Boolean functions

✔ Combined for security

General Model​

  • State:

    sts_tst​

  • Output:

    zt=h(st)z_t = h(s_t)zt​=h(st​)

  • Update:

    st+1=g(st)s_{t+1} = g(s_t)st+1​=g(st​)

Key Idea​

  • LFSR → fast but linear

  • Add:

    • Non-linear function (g)
    • Non-linear output (h)

✔ Prevents algebraic attacks

Example Structure (Typical)​

  • LFSR generates bits
  • NLFSR mixes bits
  • Output function combines both

SNOW-like Design​

  • Uses:

    • LFSR (linear state update)
    • FSM (non-linear mixing)

  • Operates in:

    GF(28)GF(2^8)GF(28)

  • Uses polynomial-based arithmetic

Grain Cipher (Example)​

Components​

  • LFSR (linear)
  • NLFSR (non-linear)

Output​

zt=h(x1,x2,...,xn)z_t = h(x_1, x_2, ..., x_n)zt​=h(x1​,x2​,...,xn​)

Initialization​

  • Run: 256 cycles256 \text{ cycles}256 cycles

✔ Before generating keystream

Functions​

  • Feedback:

    g(⋅)g(\cdot)g(⋅)

  • Output:

    h(⋅)h(\cdot)h(⋅)

✔ Both include non-linearity

Important Concepts​

1. Internal State​

sts_tst​
  • Determines entire keystream

2. Key Recovery Target​

  • Attacker tries to find: s0 or keys_0 \text{ or key}s0​ or key

3. Security​

  • Depends on:

    • Non-linearity
    • State size
    • Initialization

Attacks​

  • Algebraic attacks
  • Correlation attacks
  • Known keystream attacks

Weakness​

  • Poor design → recover: sts_tst​

→ Full keystream known

Example Cipher​

  • Helix (stream cipher design)

Summary​

  • Modern stream ciphers:

    • LFSR + NLFSR
    • Non-linear output
    • Initialization phase

✔ Designed to resist linear & algebraic attacks

Lecture 53:
Shamir secret sharing​

Secret Sharing (Threshold Scheme)​

Goal​

  • Split secret:

    SSS

  • Into shares so that:

    • Any:

      t userst \text{ users}t users

      → can recover

    • Less than (t) → cannot learn anything

Polynomial Construction​

  • Choose random polynomial:

    f(x)=S+a1x+a2x2+⋯+at−1xt−1f(x) = S + a_1 x + a_2 x^2 + \cdots + a_{t-1} x^{t-1}f(x)=S+a1​x+a2​x2+⋯+at−1​xt−1

  • Degree:

    t−1t-1t−1

Share Distribution​

  • For each user (i):
Sharei=(i,;f(i))\text{Share}_i = (i,; f(i))Sharei​=(i,;f(i))

Reconstruction (Lagrange Interpolation)​

  • Using any (t) shares:
S=f(0)S = f(0)S=f(0)

Formula​

f(0)=∑i=1tyi⋅λif(0) = \sum_{i=1}^{t} y_i \cdot \lambda_if(0)=i=1∑t​yi​⋅λi​

where

λi=∏j≠ixjxj−xi\lambda_i = \prod_{j \ne i} \frac{x_j}{x_j - x_i}λi​=j=i∏​xj​−xi​xj​​

Key Property​

  • Polynomial of degree (t-1) → uniquely determined by (t) points

✔ Security guarantee

Security Insight​

  • With fewer than (t) shares:
Infinite possible polynomials\text{Infinite possible polynomials}Infinite possible polynomials

❌ Secret cannot be guessed

Visual Intuition​

  • 1 share → many curves possible
  • (t) shares → single curve

✔ Secret fixed

Application: Group Key Recovery​

  • Secret key:

    K=f(0)K = f(0)K=f(0)

  • Each user has:

    f(i)f(i)f(i)

  • Any (t) users:

    • Combine shares
    • Recover: KKK

Broadcast Scenario​

  • Authority (GM) generates:

    f(x)f(x)f(x)

  • Sends shares to users

  • Broadcasts helper function:

    h(x)=f(x)+r(x)h(x) = f(x) + r(x)h(x)=f(x)+r(x)

User Computation​

  • Each user knows:

    f(i)f(i)f(i)

  • Using broadcast:

    h(i)h(i)h(i)

  • Can compute:

    r(i)r(i)r(i)

Key Insight​

  • Only authorized group (≥ t users):

    • Can reconstruct secret

Advantages​

  • Fault tolerance
  • No single point of failure
  • Secure distributed key management

Summary​

  • Secret → polynomial
  • Shares → points on curve
  • Reconstruction → interpolation

✔ Foundation of secure group cryptography

Lecture 54:
Identity based Encryption (IBE)​

Identity-Based Encryption (IBE)​

Idea​

  • Public key = user identity ID∈0,1∗ID \in {0,1}^*ID∈0,1∗

✔ No certificates needed

Setup​

  • Choose groups:

    G,;GTG,; G_TG,;GT​

  • Generator:

    g∈Gg \in Gg∈G

  • Master secret:

    α\alphaα

  • Public key:

    gαg^\alphagα

Hash Functions​

  • Map identity:

    H1(ID)∈GH_1(ID) \in GH1​(ID)∈G

  • Key derivation:

    H2:GT→0,1ℓH_2: G_T \rightarrow {0,1}^\ellH2​:GT​→0,1ℓ

Bilinear Pairing​

e:G×G→GTe: G \times G \rightarrow G_Te:G×G→GT​

Properties​

  1. Bilinearity:

    e(ua,;vb)=e(u,v)abe(u^a,; v^b) = e(u,v)^{ab}e(ua,;vb)=e(u,v)ab

  2. Non-degenerate:

    e(g,g)≠1e(g,g) \ne 1e(g,g)=1

  3. Efficient

Private Key Generation​

  • For user with identity (ID):
dID=H1(ID)αd_{ID} = H_1(ID)^\alphadID​=H1​(ID)α

Encryption​

  • Message:

    mmm

  • Choose random:

    rrr

Compute​

U=grU = g^rU=gr V=m⊕H2(e(H1(ID),;gα)r)V = m \oplus H_2\big(e(H_1(ID),; g^\alpha)^r\big)V=m⊕H2​(e(H1​(ID),;gα)r)

Ciphertext​

C=(U,;V)C = (U,; V)C=(U,;V)

Decryption​

  • Using private key: dIDd_{ID}dID​

Compute​

e(dID,;U)=e(H1(ID)α,;gr)e(d_{ID},; U) = e(H_1(ID)^\alpha,; g^r)e(dID​,;U)=e(H1​(ID)α,;gr) =e(H1(ID),;g)αr= e(H_1(ID),; g)^{\alpha r}=e(H1​(ID),;g)αr

Recover Message​

m=V⊕H2(e(dID,;U))m = V \oplus H_2\big(e(d_{ID},; U)\big)m=V⊕H2​(e(dID​,;U))

Correctness​

  • Encryption uses:

    e(H1(ID),;gα)re(H_1(ID),; g^\alpha)^re(H1​(ID),;gα)r

  • Decryption computes same:

    e(H1(ID),;g)αre(H_1(ID),; g)^{\alpha r}e(H1​(ID),;g)αr

✔ Values match

Key Insight​

  • Identity → public key
  • Trusted authority generates private keys

Advantages​

  • No certificates
  • Simple key management

Limitation​

  • Key escrow problem:

    • Authority knows all private keys

Summary​

  • Uses:

    • Pairing (e)
    • Hash to group
    • XOR masking

✔ Foundation: Boneh–Franklin IBE scheme

Lecture 55:
Attribute based Encryption (ABE)​

Attribute-Based Encryption (ABE)​

Idea​

  • Instead of identity → use attributes
Attributes=MSc,;Math,;Student,;...\text{Attributes} = {\text{MSc},; \text{Math},; \text{Student},; ...}Attributes=MSc,;Math,;Student,;...

✔ Access based on properties, not identity

Motivation​

  • Traditional (IBE):

    • One-to-one communication

  • ABE:

    • One-to-many communication

✔ Encrypt once → many authorized users can decrypt

Access Policy​

  • Ciphertext defined by policy:
Policy =(MSc∧Math)\text{Policy } = (\text{MSc} \land \text{Math})Policy =(MSc∧Math)
  • Only users with matching attributes can decrypt

Types of ABE​

1. Key-Policy ABE (KP-ABE)​

  • Ciphertext → attributes
  • Key → policy

2. Ciphertext-Policy ABE (CP-ABE)​

  • Ciphertext → policy
  • Key → attributes

✔ More practical

Setup​

  • Groups:

    G,;GTG,; G_TG,;GT​

  • Generator:

    ggg

  • Pairing:

    e:G×G→GTe: G \times G \rightarrow G_Te:G×G→GT​

Encryption​

  • Message:

    mmm

  • Random:

    rrr

Compute​

C1=grC_1 = g^rC1​=gr C2=m⊕H2(e(H1(attributes),;gα)r)C_2 = m \oplus H_2\big(e(H_1(\text{attributes}),; g^\alpha)^r\big)C2​=m⊕H2​(e(H1​(attributes),;gα)r)

Ciphertext​

C=(C1,;C2,;policy)C = (C_1,; C_2,; \text{policy})C=(C1​,;C2​,;policy)

Key Generation​

  • For user with attributes:

    Attru\text{Attr}_uAttru​

  • Generate secret key components

Decryption​

  • If attributes satisfy policy:

Compute pairing​

e(d,;C1)e(d,; C_1)e(d,;C1​)

Recover​

m=C2⊕H2(pairing result)m = C_2 \oplus H_2(\text{pairing result})m=C2​⊕H2​(pairing result)

Key Insight​

  • Uses bilinear property:
e(ga,;gb)=e(g,g)abe(g^a,; g^b) = e(g,g)^{ab}e(ga,;gb)=e(g,g)ab

✔ Enables policy-based access

Example​

  • Policy:

    • “MSc Math Students”

  • Users:

    • MSc Math → ✔ decrypt
    • BSc Math → ❌ cannot
    • MSc Physics → ❌ cannot

Applications​

  • Secure file sharing
  • Cloud access control
  • Broadcast encryption

Broadcast Scenario​

  • Encrypt once
  • Only authorized subset: SSS can decrypt

Advantages​

  • Fine-grained access control
  • No need for multiple encryptions

Limitation​

  • Complex key management
  • Computation overhead

Summary​

  • Identity → replaced by attributes

  • Policy-based encryption

  • Built using:

    • Pairings
    • Hash functions

✔ Powerful for real-world access control systems

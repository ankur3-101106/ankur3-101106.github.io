Lecture 21:
Advanced Encryption Standard (AES)​

Overview​

  • Modern block cipher replacing DES
  • Standardized in 2001 (NIST)
  • Based on Rijndael algorithm

Key Features​

  • Block size:

    128 bits128 \text{ bits}128 bits

  • Key sizes:

    • 128-bit → 10 rounds
    • 192-bit → 12 rounds
    • 256-bit → 14 rounds

AES Variants​

VariantKey SizeRounds
AES-12812810
AES-19219212
AES-25625614

Design Goals​

  • Resistant to:

    • Differential attacks
    • Linear attacks

  • Fast and efficient

  • Simple structure

Data Representation (State)​

  • Plaintext = 128 bits → 16 bytes

Stored as Matrix:​

[S00S01S02S03S10S11S12S13S20S21S22S23S30S31S32S33]\begin{bmatrix} S_{00} & S_{01} & S_{02} & S_{03} \\ S_{10} & S_{11} & S_{12} & S_{13} \\ S_{20} & S_{21} & S_{22} & S_{23} \\ S_{30} & S_{31} & S_{32} & S_{33} \end{bmatrix}​S00​S10​S20​S30​​S01​S11​S21​S31​​S02​S12​S22​S32​​S03​S13​S23​S33​​​
  • Each element = 1 byte (8 bits)

AES Structure​

Initial Step​

  • AddRoundKey

Main Rounds (r − 1 times)​

Each round consists of:

  1. SubBytes
  2. ShiftRows
  3. MixColumns
  4. AddRoundKey

Final Round​

  • SubBytes
  • ShiftRows
  • AddRoundKey ❌ No MixColumns

Round Flow​

Core Operations​

1. SubBytes​

  • Non-linear substitution
  • Uses S-box
Sij→S(Sij)S_{ij} \rightarrow S(S_{ij})Sij​→S(Sij​)
  • Input: 8-bit
  • Output: 8-bit

2. ShiftRows​

  • Row-wise shifting:

    • Row 0 → no shift
    • Row 1 → left shift by 1
    • Row 2 → left shift by 2
    • Row 3 → left shift by 3

3. MixColumns​

  • Column-wise transformation
  • Uses polynomial arithmetic in:
GF(28)GF(2^8)GF(28)

4. AddRoundKey​

  • XOR with round key:
State=State⊕RoundKey\text{State} = \text{State} \oplus \text{RoundKey}State=State⊕RoundKey

Key Expansion​

  • Generates:

    K1,K2,...,KrK_1, K_2, ..., K_rK1​,K2​,...,Kr​

  • Each round uses different key

Important Concepts​

  • Works on bytes (8-bit units)

  • Uses:

    • Finite field arithmetic
    • Polynomial operations

Why AES is Strong​

  • Non-linearity (S-box)
  • Diffusion (MixColumns)
  • Multiple rounds
  • Large key sizes

Key Takeaways​

  • AES = substitution + permutation based cipher
  • Uses 4 core operations
  • Works on 4×4 byte matrix
  • Most widely used encryption standard today

Final Conclusion​

  • AES is:

    • Secure
    • Efficient
    • Standard for modern encryption
Widely used in: HTTPS, Wi-Fi, Banking, etc.\text{Widely used in: HTTPS, Wi-Fi, Banking, etc.}Widely used in: HTTPS, Wi-Fi, Banking, etc.

Lecture 22:
Advanced Encryption Standard (AES) - Continued​

AES S-Box (Algebraic Construction)​

  • Not just a lookup table
  • Built using finite field ( GF(2^8) )

Steps to Compute S-Box​

  1. Represent byte as polynomial in ( GF(2^8) )
a(x)=a0+a1x+a2x2+⋯+a7x7a(x) = a_0 + a_1 x + a_2 x^2 + \cdots + a_7 x^7a(x)=a0​+a1​x+a2​x2+⋯+a7​x7
  1. Compute multiplicative inverse:
a−1 in GF(28)a^{-1} \text{ in } GF(2^8)a−1 in GF(28)
  • If ( a = 0 ), define inverse as 0
  1. Apply affine transformation:
bi=ai⊕a(i+4)⊕a(i+5)⊕a(i+6)⊕a(i+7)⊕cib_i = a_i \oplus a_{(i+4)} \oplus a_{(i+5)} \oplus a_{(i+6)} \oplus a_{(i+7)} \oplus c_ibi​=ai​⊕a(i+4)​⊕a(i+5)​⊕a(i+6)​⊕a(i+7)​⊕ci​
  • Indices mod 8
  • ( c_i ) = constant

Example​

Input:

531653_{16}5316​

Output:

ED16ED_{16}ED16​

MixColumns Operation​

  • Column-wise transformation
  • Based on matrix multiplication in ( GF(2^8) )

Transformation​

[S0′S1′S2′S3′]=[2311123111233112]⋅[S0S1S2S3]\begin{bmatrix} S'_0 \\ S'_1 \\ S'_2 \\ S'_3 \end{bmatrix} = \begin{bmatrix} 2 & 3 & 1 & 1 \\ 1 & 2 & 3 & 1 \\ 1 & 1 & 2 & 3 \\ 3 & 1 & 1 & 2 \end{bmatrix} \cdot \begin{bmatrix} S_0 \\ S_1 \\ S_2 \\ S_3 \end{bmatrix}​S0′​S1′​S2′​S3′​​​=​2113​3211​1321​1132​​⋅​S0​S1​S2​S3​​​

Notes​

  • Multiplication done in:

    GF(28)GF(2^8)GF(28)

  • Addition = XOR

Key Scheduling (Key Expansion)​

Input​

  • Initial key: 128 bits=16 bytes128 \text{ bits} = 16 \text{ bytes}128 bits=16 bytes

Output​

  • Round keys: K1,K2,...,K11K_1, K_2, ..., K_{11}K1​,K2​,...,K11​

(each 128-bit)

Word Representation​

  • Split key into 4 words:
W1,W2,W3,W4W_1, W_2, W_3, W_4W1​,W2​,W3​,W4​

(each 32-bit)

Generating Next Words​

Formula​

Wi=Wi−4⊕Wi−1W_i = W_{i-4} \oplus W_{i-1}Wi​=Wi−4​⊕Wi−1​

Special Case (every 4th word)​

Apply function ( g ):

Wi=Wi−4⊕g(Wi−1)W_i = W_{i-4} \oplus g(W_{i-1})Wi​=Wi−4​⊕g(Wi−1​)

g Function​

Steps​

  1. Rotation
[b0,b1,b2,b3]→[b1,b2,b3,b0][b_0, b_1, b_2, b_3] \rightarrow [b_1, b_2, b_3, b_0][b0​,b1​,b2​,b3​]→[b1​,b2​,b3​,b0​]
  1. SubBytes (S-box)
bi→S(bi)b_i \rightarrow S(b_i)bi​→S(bi​)
  1. Add Round Constant
bi=bi⊕Rconb_i = b_i \oplus Rconbi​=bi​⊕Rcon

Decryption (Inverse Operations)​

EncryptionDecryption
SubBytesInvSubBytes
ShiftRowsInvShiftRows
MixColumnsInvMixColumns
AddRoundKeySame (XOR)

Important Insights​

  • XOR is its own inverse: a⊕b⊕b=aa \oplus b \oplus b = aa⊕b⊕b=a

  • AES combines:

    • Field arithmetic
    • Bitwise operations

Complete Round (Concept)​

Key Takeaways​

  • S-box:

    • Built using algebra (not random)

  • MixColumns:

    • Matrix × vector in ( GF(2^8) )

  • Key schedule:

    • Expands key into round keys

  • Strong due to:

    • Non-linearity + diffusion

Final Conclusion​

  • AES internally uses:

    GF(28)GF(2^8)GF(28)

  • Combines:

    • Algebra
    • Bit operations

✔ This structured design makes AES secure and efficient

Lecture 23:
Introduction to Public Key Cryptosystem, Diffie-Hellman Key Exchange​

Motivation​

  • Two parties (Alice & Bob) communicate over insecure channel

  • Goal:

    • Ensure confidentiality

Symmetric Key Problem​

Model​

  • Same key used: Encryption key=Decryption key=K\text{Encryption key} = \text{Decryption key} = KEncryption key=Decryption key=K

Issues​

  1. Key Distribution Problem

    • How to securely share ( K )?

  2. Scalability Problem

    • For ( n ) users: n(n−1)2\frac{n(n-1)}{2}2n(n−1)​ keys required

Diffie-Hellman Key Exchange​

Purpose​

  • Establish shared secret key over insecure channel

Setup​

  • Public: p=prime,g=generatorp = \text{prime}, \quad g = \text{generator}p=prime,g=generator

Example:

p=23,;g=5p = 23,; g = 5p=23,;g=5

Step 1: Secret Selection​

  • Alice:

    a=6a = 6a=6

  • Bob:

    b=15b = 15b=15

Step 2: Compute Public Values​

A=gamod  p=56mod  23=8A = g^a \mod p = 5^6 \mod 23 = 8A=gamodp=56mod23=8 B=gbmod  p=515mod  23=19B = g^b \mod p = 5^{15} \mod 23 = 19B=gbmodp=515mod23=19

Step 3: Exchange​

  • Alice → Bob: ( A )
  • Bob → Alice: ( B )

Step 4: Shared Secret​

  • Alice computes:

    K=Bamod  p=196mod  23=2K = B^a \mod p = 19^6 \mod 23 = 2K=Bamodp=196mod23=2

  • Bob computes:

    K=Abmod  p=815mod  23=2K = A^b \mod p = 8^{15} \mod 23 = 2K=Abmodp=815mod23=2

✔ Same key derived

Security Basis​

  • Based on Discrete Logarithm Problem
gamod  p⇒a is hard to computeg^a \mod p \Rightarrow a \text{ is hard to compute}gamodp⇒a is hard to compute

Public Key Cryptosystem​

Idea​

  • Each user has two keys:
  1. Public Key (Encryption key)
  2. Private Key (Decryption key)

Communication Flow​

  1. Bob generates:

    • Public key: ( KpubK_{ pub } Kpub​ )
    • Private key: ( KprivK_{priv} Kpriv​ )

  2. Bob publishes:

    KpubK_{pub}Kpub​

  3. Alice encrypts:

    C=E(Kpub,M)C = E(K_{pub}, M)C=E(Kpub​,M)

  4. Bob decrypts:

    M=D(Kpriv,C)M = D(K_{priv}, C)M=D(Kpriv​,C)

Key Property​

  • Given: KpubK_{pub}Kpub​

❌ It should be computationally infeasible to find:

KprivK_{priv}Kpriv​

Advantages over Symmetric​

  • No need to share secret key
  • Scalable
  • Suitable for large networks

Examples of PKC​

  • RSA
  • ElGamal
  • Knapsack Cryptosystem

Key Takeaways​

  • Diffie-Hellman → key exchange
  • PKC → separate keys for encryption/decryption
  • Security → based on hard mathematical problems

Final Conclusion​

  • Public key cryptography solves:

    • Key distribution problem
    • Scalability issues

Foundation of modern secure communication (HTTPS, SSH, etc.)\text{Foundation of modern secure communication (HTTPS, SSH, etc.)}Foundation of modern secure communication (HTTPS, SSH, etc.)

Lecture 24:
Knapsack Cryptosystem​

AKA Subset Sum​

Problem Definition​

Given:

  • Numbers: a1,a2,...,ana_1, a_2, ..., a_na1​,a2​,...,an​
  • Target sum: SSS

Find binary values xi∈0,1x_i \in {0,1}xi​∈0,1 such that:

∑i=1naixi=S\sum_{i=1}^{n} a_i x_i = Si=1∑n​ai​xi​=S

Complexity​

  • Total subsets:

    2n2^n2n

  • Brute force → Exponential time (hard problem)

Superincreasing Sequence​

Definition​

Sequence where:

ai>∑j=1i−1aja_i > \sum_{j=1}^{i-1} a_jai​>j=1∑i−1​aj​

Property​

  • Subset sum becomes easy (polynomial time)
  • Solve using greedy approach

Greedy Solution Idea​

  • Start from largest element
  • If: S≥anS \ge a_nS≥an​ → include it
  • Reduce: S=S−anS = S - a_nS=S−an​
  • Repeat

Knapsack Cryptosystem Idea​

Step 1: Private Key (Bob)​

  • Choose superincreasing sequence:

    a1,a2,...,ana_1, a_2, ..., a_na1​,a2​,...,an​

  • Choose:

    m>∑aim > \sum a_im>∑ai​

  • Choose:

    w such that gcd⁡(w,m)=1w \text{ such that } \gcd(w,m)=1w such that gcd(w,m)=1

Step 2: Public Key​

Transform:

Ai=w⋅aimod  mA_i = w \cdot a_i \mod mAi​=w⋅ai​modm
  • Public key: A1,A2,...,An{A_1, A_2, ..., A_n}A1​,A2​,...,An​

Step 3: Encryption (Alice)​

  • Message bits:

    x1,x2,...,xnx_1, x_2, ..., x_nx1​,x2​,...,xn​

  • Ciphertext:

C=∑AixiC = \sum A_i x_iC=∑Ai​xi​

Step 4: Decryption (Bob)​

  • Compute:
S=C⋅w−1mod  mS = C \cdot w^{-1} \mod mS=C⋅w−1modm
  • Now solve:
S=∑aixiS = \sum a_i x_iS=∑ai​xi​

✔ Easy because sequence is superincreasing

Key Insight​

  • Public sequence looks random (hard problem)
  • Private sequence is structured (easy problem)

Why It Works​

  • Transformation hides structure:

    Ai=waimod  mA_i = w a_i \mod mAi​=wai​modm

  • Only Bob knows:

    • w−1w^{-1}w−1
    • original sequence

Security Idea​

  • Based on:

    • Hardness of subset sum problem

Weakness​

  • Original knapsack cryptosystem:

    • Broken (not secure today)

  • Structure can be recovered

Key Takeaways​

  • Subset sum:

    • Hard in general
    • Easy for superincreasing sequence

  • Cryptosystem:

    • Uses transformation to hide structure

Final Conclusion​

  • Knapsack cryptosystem introduced idea of:

    • trapdoor function
Easy with secret, hard without it\text{Easy with secret, hard without it}Easy with secret, hard without it

✔ Important concept in public key cryptography

Lecture 25:
RSA Cryptosystem​

Overview​

  • Public key cryptosystem by Rivest, Shamir, Adleman
  • Based on: Integer factorization problem\text{Integer factorization problem}Integer factorization problem

Key Idea​

  • Two keys:

    • Public key → encryption
    • Private key → decryption

Key Generation (Bob)​

Step 1: Choose Primes​

p,q are large primesp, q \text{ are large primes}p,q are large primes

Step 2: Compute​

n=p⋅qn = p \cdot qn=p⋅q ϕ(n)=(p−1)(q−1)\phi(n) = (p-1)(q-1)ϕ(n)=(p−1)(q−1)

Step 3: Choose Public Exponent​

e such that gcd⁡(e,ϕ(n))=1e \text{ such that } \gcd(e,\phi(n))=1e such that gcd(e,ϕ(n))=1

Step 4: Compute Private Key​

d≡e−1(modϕ(n))d \equiv e^{-1} \pmod{\phi(n)}d≡e−1(modϕ(n))

i.e.

e⋅d≡1(modϕ(n))e \cdot d \equiv 1 \pmod{\phi(n)}e⋅d≡1(modϕ(n))

Keys​

  • Public key:

    (e,n)(e,n)(e,n)

  • Private key:

    (d,n)(d,n)(d,n)

Encryption (Alice)​

Message:

m∈Znm \in \mathbb{Z}_nm∈Zn​

Ciphertext:

C=memod  nC = m^e \mod nC=memodn

Decryption (Bob)​

m=Cdmod  nm = C^d \mod nm=Cdmodn

Why It Works​

med≡m(modn)m^{ed} \equiv m \pmod{n}med≡m(modn)

Using:

  • Euler’s theorem
  • ed≡1(modϕ(n))ed \equiv 1 \pmod{\phi(n)}ed≡1(modϕ(n))

Example​

Step 1​

p=7,;q=17p = 7,; q = 17p=7,;q=17

Step 2​

n=7⋅17=119n = 7 \cdot 17 = 119n=7⋅17=119 ϕ(n)=6⋅16=96\phi(n) = 6 \cdot 16 = 96ϕ(n)=6⋅16=96

Step 3​

Choose:

e=5e = 5e=5

Step 4​

Find:

d≡e−1(mod96)d \equiv e^{-1} \pmod{96}d≡e−1(mod96) 5d≡1(mod96)5d \equiv 1 \pmod{96}5d≡1(mod96) d=77d = 77d=77

(because 5⋅77=385=4⋅96+15 \cdot 77 = 385 = 4 \cdot 96 + 15⋅77=385=4⋅96+1)

Final Keys​

  • Public:

    (5,119)(5,119)(5,119)

  • Private:

    (77,119)(77,119)(77,119)

Important Notes​

1. Large Primes Required​

  • Small primes → easy to factor
  • Security depends on: n=p⋅qn = p \cdot qn=p⋅q

2. Prime Testing​

  • Algorithms:

    • Miller-Rabin

3. Efficient Exponentiation​

Compute:

abmod  na^b \mod nabmodn

Using:

  • Fast exponentiation (square & multiply)

Key Property (Flexibility)​

  • RSA supports:

    • Encryption
    • Digital signatures

Key Takeaways​

  • Based on:

    • Factorization hardness

  • Uses:

    • Modular arithmetic
    • Euler’s theorem

  • Requires:

    • Large primes

Final Conclusion​

  • RSA is one of the most important:
Public key cryptosystems in practice\text{Public key cryptosystems in practice}Public key cryptosystems in practice

✔ Used in:

  • HTTPS
  • Digital signatures
  • Secure communication

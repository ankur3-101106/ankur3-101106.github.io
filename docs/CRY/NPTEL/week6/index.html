Lecture 26:
More on RSA​

Fast Modular Exponentiation (Square & Multiply)​

Goal​

Compute:

anmod  ma^n \mod manmodm

efficiently

Idea​

  • Write exponent in binary:

    n=(bkbk−1...b0)2n = (b_k b_{k-1} ... b_0)_2n=(bk​bk−1​...b0​)2​

  • Then:

    an=ab020⋅ab121⋯abk2ka^n = a^{b_0 2^0} \cdot a^{b_1 2^1} \cdots a^{b_k 2^k}an=ab0​20⋅ab1​21⋯abk​2k

Key Observation​

a2i=(a2i−1)2a^{2^i} = (a^{2^{i-1}})^2a2i=(a2i−1)2

✔ Compute using repeated squaring

Complexity​

  • Number of steps: O(log⁡n)O(\log n)O(logn)

✔ Much faster than naive ( O(n) )

Algorithm (Concept)​

  1. Convert ( n ) to binary

  2. Initialize result = 1

  3. For each bit:

    • Square current value
    • If bit = 1 → multiply

Side-Channel Attack (Timing Attack)​

Problem​

  • Computation time depends on:

    • Bits of exponent

Idea​

  • Attacker measures:

    • Execution time

  • Infers:

    • Whether bit = 1 or 0

Example Insight​

  • If extra multiplication occurs → bit = 1
  • Leads to key leakage

RSA Key Generation (Practical Issues)​

1. Prime Generation​

Need:

p,q large primes (e.g., 512+ bits)p, q \text{ large primes (e.g., 512+ bits)}p,q large primes (e.g., 512+ bits)

Naive Method (Inefficient)​

  • Check divisibility up to: n\sqrt{n}n​

❌ Too slow for large numbers

Sieve Method​

  • Remove multiples:

    • 2, 3, 5, ...

✔ Works for small ( n ) ❌ Not practical for large primes

Efficient Approach​

  • Use probabilistic tests

Example:​

  • Miller-Rabin test

✔ Fast ✔ High probability of correctness

RSA Security​

Based On​

n=p⋅qn = p \cdot qn=p⋅q

  • Public:

    nnn

  • Secret:

    p,qp, qp,q

Hard Problem​

Factorization of large integers\text{Factorization of large integers}Factorization of large integers

Why Secure?​

  • Given: nnn

❌ Hard to compute:

p,qp, qp,q

Key Insight​

  • If attacker finds: p,qp, qp,q

→ Can compute:

ϕ(n)\phi(n)ϕ(n)

→ Then:

ddd

→ System breaks

Key Takeaways​

  • Square & multiply:

    • Efficient exponentiation

  • Timing attacks:

    • Real-world vulnerability

  • Prime generation:

    • Uses probabilistic tests

  • RSA security:

    • Depends on factorization hardness

Final Conclusion​

  • RSA efficiency:

    O(log⁡n)O(\log n)O(logn)

  • RSA security:

    Hardness of factoring n=p⋅q\text{Hardness of factoring } n = p \cdot qHardness of factoring n=p⋅q

✔ Still widely used, but requires:

  • Large keys
  • Secure implementations

Lecture 27:
Primality Testing​

Goal​

  • Check whether a number ( n ) is prime or composite

Fermat Primality Test​

Idea (from Fermat’s Little Theorem)​

If ( p ) is prime and ( \gcd(a,p)=1 ), then:

ap−1≡1(modp)a^{p-1} \equiv 1 \pmod{p}ap−1≡1(modp)

Algorithm​

  1. Choose random:

    a<na < na<n

  2. Compute:

    y=an−1mod  ny = a^{n-1} \mod ny=an−1modn

  3. Check:

  • If:

    y≠1y \ne 1y=1

    → ( n ) is composite ✅

  • If:

    y=1y = 1y=1

    → ( n ) is probably prime

Limitation​

  • Some composite numbers pass test

  • Called:

    • Fermat pseudoprimes

Miller–Rabin Primality Test​

Stronger Test​

Step 1: Write​

n−1=2k⋅mn - 1 = 2^k \cdot mn−1=2k⋅m

where:

m is oddm \text{ is odd}m is odd

Step 2: Choose Random Base​

a<na < na<n

Step 3: Compute​

b0=ammod  nb_0 = a^m \mod nb0​=ammodn

Step 4: Check Conditions​

  • If: b0≡1(modn)b_0 \equiv 1 \pmod{n}b0​≡1(modn) → probably prime
  • Else compute:
bi=bi−12mod  nb_i = b_{i-1}^2 \mod nbi​=bi−12​modn

Check if any:

bi≡−1(modn)b_i \equiv -1 \pmod{n}bi​≡−1(modn)

→ probably prime

Final Decision​

  • If none satisfy: ±1\pm 1±1

→ ( n ) is composite ✅

Key Property​

  • If test says composite → always correct
  • If says prime → probabilistic

Why Better than Fermat​

  • Detects:

    • Carmichael numbers

  • Much lower error probability

Key Takeaways​

  • Fermat test:

    • Fast but weak

  • Miller-Rabin:

    • Fast and reliable

  • Used in:

    • RSA key generation

Final Conclusion​

  • Primality testing is:

    Essential for generating large primes\text{Essential for generating large primes}Essential for generating large primes

  • Practical systems use:

    • Miller-Rabin test

✔ Efficient + high confidence for cryptography

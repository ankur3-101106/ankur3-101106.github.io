Lecture 36:
Message Authentication​

Message Authentication (Basic Idea)​

  • Goal: verify message is from sender

Using Symmetric Key​

  • Alice sends: (M,;EK(H(M)))(M,; E_K(H(M)))(M,;EK​(H(M)))

At Receiver​

  • Compute:

    H(M)H(M)H(M)

  • Decrypt:

    DK(y)=H(M)D_K(y) = H(M)DK​(y)=H(M)

  • Compare

✔ If equal → authenticated

Issue​

  • Receiver can forge message
  • No proof of origin

Hash Function​

  • Input: arbitrary size

  • Output:

    fixed size digest\text{fixed size digest}fixed size digest

  • Public (no key)

MAC (Message Authentication Code)​

  • Uses secret key
MAC=EK(H(M))\text{MAC} = E_K(H(M))MAC=EK​(H(M))

Limitation​

  • Protects from third party
  • Not from malicious receiver

Digital Signature​

  • Uses private key

Signing​

y=DA(M)y = D_{A}(M)y=DA​(M)

Verification​

M=EA(y)M = E_{A}(y)M=EA​(y)

✔ Only sender can sign

Property​

  • Non-repudiation
  • Cannot be forged by receiver

Public Key Directory​

  • Stores:

    • Public keys of users

Authentication + Confidentiality (Hybrid)​

Step 1: Sign​

y=DA(M)y = D_A(M)y=DA​(M)

Step 2: Encrypt​

C=EK(M,y)C = E_K(M, y)C=EK​(M,y)

Receiver​

  • Decrypt:

    (M,y)(M, y)(M,y)

  • Verify:

    M=EA(y)M = E_A(y)M=EA​(y)

Alternative (Pure Public Key)​

Sender​

  • Sign:

    y=DA(M)y = D_A(M)y=DA​(M)

  • Encrypt:

    C=EB(M,y)C = E_B(M, y)C=EB​(M,y)

Receiver​

  • Decrypt:

    (M,y)(M, y)(M,y)

  • Verify:

    M=EA(y)M = E_A(y)M=EA​(y)

Final Properties​

  • Confidentiality:

    EB(⋅)E_B(\cdot)EB​(⋅)

  • Authentication:

    DA(⋅)D_A(\cdot)DA​(⋅)

Key Insight​

  • Signature → proves sender
  • Encryption → hides message

Conclusion​

  • Combine:

    • Hash + MAC → basic authentication
    • Digital signature → strong authentication
    • Encryption → confidentiality

✔ Used together in secure communication

Lecture 37:
Digital Signature​

Purpose​

  • Authentication
  • Integrity
  • Non-repudiation

Basic Idea​

  • Sender signs using private key
  • Receiver verifies using public key

RSA Signature​

Key Setup​

n=p⋅qn = p \cdot qn=p⋅q ϕ(n)=(p−1)(q−1)\phi(n) = (p-1)(q-1)ϕ(n)=(p−1)(q−1) e⋅d≡1(modϕ(n))e \cdot d \equiv 1 \pmod{\phi(n)}e⋅d≡1(modϕ(n))

Signing​

y=mdmod  ny = m^d \mod ny=mdmodn

Verification​

m′=yemod  nm' = y^e \mod nm′=yemodn

Valid if:

m′=mm' = mm′=m

Correctness​

ed=kϕ(n)+1ed = k\phi(n) + 1ed=kϕ(n)+1 ye=(md)e=med=mkϕ(n)+1y^e = (m^d)^e = m^{ed} = m^{k\phi(n)+1}ye=(md)e=med=mkϕ(n)+1 =(mϕ(n))k⋅m≡1k⋅m≡m(modn)= (m^{\phi(n)})^k \cdot m \equiv 1^k \cdot m \equiv m \pmod{n}=(mϕ(n))k⋅m≡1k⋅m≡m(modn)

Signature Scheme (Formal)​

  • Tuple:

    (P,S,K,Sign,Verify)(P, S, K, \text{Sign}, \text{Verify})(P,S,K,Sign,Verify)

  • PPP → plaintext space

  • SSS → signature space

  • KKK → key space

Functions​

Signk(m)→s\text{Sign}_k(m) \rightarrow sSignk​(m)→s Verifyk(m,s)→true,false\text{Verify}_k(m, s) \rightarrow {\text{true}, \text{false}}Verifyk​(m,s)→true,false

Property​

Verifyk(m,Signk(m))=true\text{Verify}_k(m, \text{Sign}_k(m)) = \text{true}Verifyk​(m,Signk​(m))=true

ElGamal Signature​

Setup​

  • Choose prime:

    ppp

  • Generator:

    α\alphaα

  • Private key:

    aaa

  • Public key:

    β=αamod  p\beta = \alpha^a \mod pβ=αamodp

Signing​

Choose random:

k such that gcd⁡(k,p−1)=1k \text{ such that } \gcd(k, p-1)=1k such that gcd(k,p−1)=1 γ=αkmod  p\gamma = \alpha^k \mod pγ=αkmodp δ=(x−aγ)⋅k−1mod  (p−1)\delta = (x - a\gamma)\cdot k^{-1} \mod (p-1)δ=(x−aγ)⋅k−1mod(p−1)
  • Signature: (γ,δ)(\gamma, \delta)(γ,δ)

Verification​

βγ⋅γδ≡αxmod  p\beta^\gamma \cdot \gamma^\delta \equiv \alpha^x \mod pβγ⋅γδ≡αxmodp

Correctness​

βγ⋅γδ=(αa)γ⋅(αk)δ=αaγ+kδ\beta^\gamma \cdot \gamma^\delta = (\alpha^a)^\gamma \cdot (\alpha^k)^\delta = \alpha^{a\gamma + k\delta}βγ⋅γδ=(αa)γ⋅(αk)δ=αaγ+kδ kδ≡x−aγ(modp−1)k\delta \equiv x - a\gamma \pmod{p-1}kδ≡x−aγ(modp−1) aγ+kδ≡x(modp−1)a\gamma + k\delta \equiv x \pmod{p-1}aγ+kδ≡x(modp−1) ⇒αaγ+kδ≡αxmod  p\Rightarrow \alpha^{a\gamma + k\delta} \equiv \alpha^x \mod p⇒αaγ+kδ≡αxmodp

Key Points​

  • RSA → based on factorization
  • ElGamal → based on discrete log
  • Signing uses private key
  • Verification uses public key

Final Insight​

  • Signature ensures:

    • Message origin
    • Message integrity
    • Sender accountability

Lecture 38:
Key Management​

Key Management​

  • Goal: securely obtain receiver’s public key before communication

Problem​

  • Alice needs Bob’s public key eBe_BeB​
  • How to get it securely?

Public Key Distribution Methods​

1. Public Announcement​

  • Users broadcast their public keys

Issue:

  • Vulnerable to impersonation (attacker can replace key)

2. Public Key Directory (PKD)​

  • Central directory storing:

    • User ID
    • Public key

  • Sender:

    • Looks up receiver’s key
    • Encrypts message

Issue:

  • If directory is public → can be modified
  • Needs protection

3. Trusted Public Key Authority​

  • Central trusted authority maintains keys

  • Process:

    • User requests receiver’s key from authority
    • Authority provides authentic key

Issue:

  • Bottleneck (handles all requests)
  • High communication overhead

Solution → Public Key Certificates​

  • Authority issues certificate containing:

    • User ID
    • Public key
    • Timestamp
    • Digital signature of authority

Certificate Creation​

For user ( B ):

CB=Edauth(T,;IDB,;eB)C_B = E_{d_{auth}}(T,; ID_B,; e_B)CB​=Edauth​​(T,;IDB​,;eB​)
  • Signed using authority’s private key

Certificate Verification​

Alice verifies Bob’s certificate:

Deauth(CB)→(T,;IDB,;eB)D_{e_{auth}}(C_B) \rightarrow (T,; ID_B,; e_B)Deauth​​(CB​)→(T,;IDB​,;eB​)

  • Check:

    • Timestamp valid
    • ID matches
    • Signature valid

✔ Then trust ( e_B )

Communication Using Certificate​

  1. Bob sends certificate ( C_B )
  2. Alice verifies it
  3. Extracts ( e_B )
  4. Encrypts message using ( e_B )

Advantages​

  • No need to contact authority every time
  • Prevents key tampering
  • Scalable

Certificate Update​

  • If key compromised:

    • User generates new key pair
    • Requests new certificate
    • Authority signs new certificate

Certificate Contains​

  • Timestamp
  • User ID
  • Public key
  • Authority signature

X.509 Certificate Standard​

  • Widely used format

Fields:​

  • Version
  • Serial number
  • Signature algorithm
  • Issuer (authority)
  • Validity period
  • Subject (user)
  • Public key
  • Extensions
  • Digital signature

Key Insight​

  • Trust shifts from:

    • “Anyone can publish key” ❌
    • To “Authority verifies identity” ✔

Final Flow​

  • Authority signs → certificate trusted
  • User verifies → gets correct public key
  • Secure communication enabled

Lecture 39:
Key Exchange​

Key Exchange (Goal)​

  • Share symmetric key (k) over insecure channel
  • Needed because symmetric crypto (DES/AES) is faster

Diffie–Hellman (Recap)​

  • Public:

    g,;pg,; pg,;p

  • Secrets:

    a,;ba,; ba,;b

  • Exchange:

    A=gamod  p,B=gbmod  pA = g^a \mod p,\quad B = g^b \mod pA=gamodp,B=gbmodp

  • Shared key:

    K=gabmod  pK = g^{ab} \mod pK=gabmodp

Problem → Active Attack (MITM)​

  • Attacker (Oscar) intercepts messages

  • Replaces:

    • (g^a, g^b) with (g^c)

Result​

  • Alice computes:

    K1=gacK_1 = g^{ac}K1​=gac

  • Bob computes:

    K2=gbcK_2 = g^{bc}K2​=gbc

  • Oscar knows both:

    K1,;K2K_1,; K_2K1​,;K2​

✔ Can decrypt + re-encrypt messages

Using Public Key for Key Exchange​

Basic Scheme​

  1. Alice sends:

    (IDA,;eA)(ID_A,; e_A)(IDA​,;eA​)

  2. Bob chooses:

    kkk

  3. Bob sends:

    EeA(k)E_{e_A}(k)EeA​​(k)

  4. Alice decrypts:

    k=DdA(y)k = D_{d_A}(y)k=DdA​​(y)

✔ Shared key established

Problem → Still Vulnerable​

  • Oscar replaces Alice’s public key
  • Bob encrypts (k) using Oscar’s key
  • Oscar decrypts → learns (k)
  • Re-encrypts for Alice

✔ Key leaked

Secure Key Exchange (Authenticated)​

Step 1​

Alice → Bob:

EeB(N1,;IDA)E_{e_B}(N_1,; ID_A)EeB​​(N1​,;IDA​)

Step 2​

Bob → Alice:

EeA(N1,;N2)E_{e_A}(N_1,; N_2)EeA​​(N1​,;N2​)

Step 3​

Alice → Bob:

EeB(N2)E_{e_B}(N_2)EeB​​(N2​)

✔ Mutual authentication using nonces

Step 4 (Key Transfer)​

Bob sends:

EeA(k)E_{e_A}(k)EeA​​(k)

Then signs:

EdB(EeA(k))E_{d_B}(E_{e_A}(k))EdB​​(EeA​​(k))

Final Decryption (Alice)​

  1. Verify signature:

    DeB(⋅)D_{e_B}(\cdot)DeB​​(⋅)

  2. Decrypt:

    k=DdA(⋅)k = D_{d_A}(\cdot)k=DdA​​(⋅)

Why Secure?​

  • Nonces → prevent replay
  • Encryption → confidentiality
  • Signature → authentication

✔ Stops active attacker

Key Insight​

  • Diffie–Hellman alone ❌ (no authentication)
  • Public key alone ❌ (key substitution attack)
  • Combined (auth + encryption) ✔ secure

Final Flow​

  • Authenticate → verify identity
  • Exchange key → securely
  • Use symmetric key for communication

Lecture 40:
Hash Function​

Hash Function (Concept)​

  • Map large key space → fixed size table

    h:U→0,1,…,m−1h: U \rightarrow {0,1,\dots,m-1}h:U→0,1,…,m−1

  • Operations supported:

    • Insert
    • Delete
    • Search

Hash Table​

  • Size:

    mmm

  • Keys:

    k1,k2,…,knk_1, k_2, \dots, k_nk1​,k2​,…,kn​

  • Load factor:

    α=nm\alpha = \frac{n}{m}α=mn​

Collision​

  • When:

    h(k1)=h(k2)h(k_1) = h(k_2)h(k1​)=h(k2​)

  • Happens because:

    ∣U∣≫m|U| \gg m∣U∣≫m

Good Hash Function​

  • Uniform distribution: P(h(k)=j)=1mP(h(k)=j) = \frac{1}{m}P(h(k)=j)=m1​

Division Method​

h(k)=kmod  mh(k) = k \mod mh(k)=kmodm

  • Choose:

    m as primem \text{ as prime}m as prime

  • Avoid:

    m=2rm = 2^rm=2r

Multiplication Method​

  • Formula:

    h(k)=⌊m⋅(kAmod  1)⌋h(k) = \lfloor m \cdot (kA \mod 1) \rfloorh(k)=⌊m⋅(kAmod1)⌋

  • Or binary form (practical):

    h(k)=right-shift(k⋅A,;w−r)h(k) = \text{right-shift}(k \cdot A,; w-r)h(k)=right-shift(k⋅A,;w−r)

Collision Resolution​

1. Chaining​

  • Store list at each slot

2. Open Addressing​

  • No extra memory

Linear Probing​

hi(k)=(h(k)+i)mod  mh_i(k) = (h(k) + i) \mod mhi​(k)=(h(k)+i)modm

  • Problem:

    • Clustering

Double Hashing​

hi(k)=(h1(k)+i⋅h2(k))mod  mh_i(k) = (h_1(k) + i \cdot h_2(k)) \mod mhi​(k)=(h1​(k)+i⋅h2​(k))modm
  • Better distribution
  • Reduces clustering

Deletion Issue​

  • Cannot mark slot empty directly

  • Use:

    • Deleted marker

Performance​

  • Unsuccessful search (approx):

    11−α\frac{1}{1-\alpha}1−α1​

  • Examples:

    • If α=0.5\alpha = 0.5α=0.5 → ~2 probes
    • If α=0.9\alpha = 0.9α=0.9 → ~10 probes

Key Insight​

  • Small α\alphaα → faster search
  • Good hash → uniform distribution
  • Double hashing > linear probing
